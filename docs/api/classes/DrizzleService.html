方法

getDrizzle

  • getDrizzle(
        options: DrizzleModuleConfig,
        tenantId: string,
    ): Promise<PostgresJsDatabase<Record<string, unknown>> & {}>

  • 获取租户专用的 Drizzle 实例

    机制:

    1. 参数验证
    • 检查数据库连接配置的完整性
    • 确保 postgres.url 必须存在
    1. 连接池管理
    • 通过 TenantConnectionPool 获取或创建租户专用连接池
    • 每个租户使用独立的 schema 和连接池
    • 复用已存在的连接以提高性能

    原理:

    • 利用 PostgreSQL 的 schema 实现多租户隔离
    • 通过连接池复用减少资源开销
    • 基于租户ID动态路由到对应的数据库连接

    参数

    • options: DrizzleModuleConfig

      Drizzle模块配置对象

    • tenantId: string

      租户标识

    返回 Promise<PostgresJsDatabase<Record<string, unknown>> & {}>

    返回租户专用的Drizzle实例
