Class TenantMiddleware

租户中间件

主要机制:

  1. 请求拦截
  • 拦截所有HTTP请求
  • 从请求头中提取租户信息
  • 将租户信息存储到请求上下文
  1. 租户隔离
  • 使用ClsService维护请求级别的租户上下文
  • 通过TenantContextService管理租户状态
  • 支持多租户数据隔离
  1. 路径排除
  • 维护排除路径列表
  • 对特定路径使用默认租户

要点:

  • 统一的租户信息提取和验证
  • 灵活的路径排除机制
  • 完善的错误处理和日志记录

原理:

  • 基于NestJS中间件机制实现请求拦截
  • 利用ClsService实现请求作用域的数据存储
  • 通过正则表达式验证租户ID格式
